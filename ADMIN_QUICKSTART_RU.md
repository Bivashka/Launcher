# BivLauncher Admin - Быстрый старт

## 1) Первичный запуск

1. Открой страницу админки.
2. Создай root-администратора (логин + пароль).
3. После входа открой раздел `Профили и серверы`.

## 2) Базовая последовательность настройки

### Шаг 1. Создай профиль
- Название профиля (например, `Main Survival`)
- Slug (например, `main-survival`)
- RAM и JVM/Game args по умолчанию
- При необходимости загрузи иконку

### Шаг 2. Создай сервер
- Выбери профиль
- Укажи адрес и порт
- Укажи тип загрузчика (`vanilla/forge/fabric/...`) и версию MC
- При необходимости заполни RU-прокси поля и main/ru jar path

### Шаг 3. Проверь списки
- Внизу страницы проверь, что профиль и сервер появились в списках
- При необходимости нажми `Изменить` и поправь параметры

## 3) Что делать дальше

- Раздел `Сборка и рантайм`: загрузка runtime, ретеншен, косметика.
- Раздел `Интеграции`: auth provider, S3/Local storage, Discord.
- Раздел `Крэши`: просмотр и фильтрация crash-логов.
- Раздел `Документация`: статьи и FAQ для команды.

## 4) Типовые проблемы

### Нет доступа к API
Проверь:
- что контейнер `api` в статусе `Up`
- что `API_PORT=8080` доступен
- что в `.env` корректные значения `POSTGRES_*`
- что браузер открывает `http://<host>:8080/health`

### API падает на старте (Postgres auth failed)
Это значит, что пароль в контейнере БД не совпадает с `.env`.
Синхронизируй пароль пользователя `postgres` и перезапусти `api`.

## 5) Минимум по безопасности

- Сразу смени пароль администратора на длинный.
- Не оставляй дефолтные `JWT_SECRET` и `POSTGRES_PASSWORD`.
- Ограничь доступ к портам 8080/5432 через firewall.
